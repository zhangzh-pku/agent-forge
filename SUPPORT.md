# Support

## Getting Help

- Usage and setup questions: open a GitHub Discussion or Issue.
- Bug reports: use the Bug Report issue template.
- Feature requests: use the Feature Request issue template.

## Security Issues

Do not open public issues for vulnerabilities.
Follow the process in `SECURITY.md`.

## What to Include

When asking for help, include:

- Commit SHA or release version
- Runtime mode (`local` or `aws`)
- Relevant environment variables (redacted)
- Reproduction steps and expected vs actual behavior
- Logs or error output

## Response Expectations

Best-effort support is provided by maintainers and contributors.
For production-critical incidents, maintain your own on-call and rollback plan.
